SSH keys (Git over SSH)

To push and pull over SSH (default port 2222), register a public key. If you only use , you can skip this and use your account password with Git instead.

Step 1 — Generate an SSH key

ED25519 is recommended. Skip if you already use a key pair.

{{ sshKeygenCmd }}

Step 2 — Copy your public key

Print the .pub file and copy its contents. 

{{ catPubCmd }}

Step 3 — Add the key in OriginHub

Open Settings and paste the key under SSH Keys, or use your deployment’s API if you automate onboarding.

Step 4 — Clone over SSH

Example using the same SSH clone URL as in the repository Code dialog in the app. 

{{ sshCloneLine() }}

HTTP / HTTPS Git

Smart Git is served on the same origin as the REST API under /git/<owner>/<repo>. Git sends HTTP Basic auth: use your OriginHub username and account password when prompted.

Path tip: Do not add a .git suffix in the URL path — storage resolution appends it on the server. With TLS in front of the app, use https:// instead of http://. 
{{ httpsCloneLine() }}

Smart HTTP can be turned off with originhub.http.enabled=false on the server — then use SSH only.
