Documentation

Get started with OriginHub. Clone over HTTPS or SSH (JGit smart HTTP and SSH), configure your environment, and push code.

1 SSH keys (for Git over SSH)

To clone and push over SSH (default port 2222), add an SSH key to your account. If you only use HTTP or HTTPS Git, you can skip this and sign in with your username and password instead.

Step 1 — Generate SSH key

Create an ED25519 key pair. Skip this if you already have one. 

ssh-keygen -t ed25519 -C "your-email@example.com"

Step 2 — Copy your public key

Display your public key and copy it to the clipboard.

cat ~/.ssh/id_ed25519.pub

Step 4 — Clone over SSH (full URL)

git clone ssh://git@localhost:2222/owner/repo.git

2 HTTP / HTTPS Git

OriginHub exposes Git smart HTTP on the same origin as the REST API, under /git/<owner>/<repo>. Use HTTP Basic authentication: your OriginHub username and account password when Git prompts for credentials (for example on git clone, git pull, or git push).

Use the URL form below (no .git suffix in the path). With TLS in front of the app, use https:// instead of http://. 

git clone http://localhost:8080/git/owner/repo

Smart HTTP can be disabled server-side via originhub.http.enabled=false; if so, use SSH only.

3 SSH config (optional)

Alternative: you can add an entry to ~/.ssh/config for a shorter alias. 

Host originhub-local
    HostName localhost
    Port 2222
    User git
    IdentityFile ~/.ssh/id_ed25519

Then clone and push using the short host alias:

git clone originhub-local:username/repo-name.git
cd repo-name
git push origin main

4 Clone & push quick reference

Action Command
Clone (HTTPS / HTTP) git clone http://localhost:8080/git/owner/repo
Clone (SSH full URL) git clone ssh://git@localhost:2222/owner/repo.git
Clone (SSH with config) git clone originhub-local:owner/repo.git
Push to branch git push origin main
Push new branch git push -u origin feature-branch
Pull latest git pull origin main
